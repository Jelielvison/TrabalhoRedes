Topologia

Todos os dispositivos conectam-se a um único ponto central, que pode ser um hub ou switch. Quando um dispositivo deseja transmitir dados, ele envia os dados para o ponto central, que os encaminha para o dispositivo de destino. Essa topologia facilita a identificação e resolução de problemas, pois cada conexão é independente.

Todos os dispositivos compartilham o mesmo barramento de comunicação. Quando um dispositivo transmite dados, esses dados são enviados por todo o barramento. Cada dispositivo possui um endereço único, e somente o dispositivo com o endereço correspondente aceita os dados. Isso é conhecido como acesso ao meio compartilhado.

Cada dispositivo está conectado aos seus dois vizinhos mais próximos, formando um anel fechado. Quando um dispositivo transmite dados, esses dados circulam pelo anel até atingir o destino. Cada dispositivo regenera os sinais, garantindo que os dados continuem a circular pelo anel.

Cada dispositivo está conectado a todos os outros, proporcionando vários caminhos para a comunicação. Se um caminho estiver indisponível, os dados podem ser roteados por caminhos alternativos. Isso aumenta a redundância e a confiabilidade da rede.

Os dispositivos são organizados em uma estrutura hierárquica de árvore. Existem níveis, e a comunicação segue os ramos da árvore até atingir o destino. Essa topologia é eficaz para redes organizadas, como redes de empresa.

Combina duas ou mais topologias em uma única rede. Cada segmento da rede pode ter sua própria topologia, proporcionando flexibilidade. Por exemplo, uma parte da rede pode ser em estrela, enquanto outra em anel.

Cada dispositivo está conectado diretamente a outro dispositivo. A comunicação ocorre ponto a ponto, sem a necessidade de dispositivos intermediários. Isso simplifica a arquitetura da rede e é comumente encontrado em redes pequenas ou conexões diretas.

Modelos

  • Camada Física: Lida com a transmissão física dos dados sobre o meio de comunicação. Isso inclui especificações como voltagens, taxas de transmissão, conectores físicos.
  • Camada de Enlace de Dados: Responsável pelo estabelecimento de conexões lógicas entre dispositivos na mesma rede física. Ela trata erros na camada física, garantindo uma comunicação confiável.
  • Camada de Rede: Gerencia o roteamento dos dados através da rede. Decide o melhor caminho para os dados viajarem de acordo com condições como tráfego, velocidade e topologia da rede.
  • Camada de Transporte: Fornecer serviços de transporte de dados confiável, estabelecendo, mantendo e encerrando conexões. Também gerencia controle de fluxo, segmentação e retransmissão.
  • Camada de Sessão: Gerencia as sessões ou diálogos entre as aplicações. Controla o diálogo, garantindo que os dados sejam transmitidos de forma ordenada e sem erros.
  • Camada de Apresentação: Responsável pela tradução, compressão e criptografia dos dados. Garante que os dados sejam apresentados da maneira correta, independentemente dos formatos utilizados pelas aplicações.
  • Camada de Aplicação: É a camada mais próxima do usuário final. Fornece interfaces para os serviços de rede e é onde as aplicações de software interagem diretamente com o usuário.

  1. Camada de Enlace:

    • Responsável pela transmissão de dados entre dispositivos na mesma rede física.
    • Lida com endereçamento físico (por exemplo, endereços MAC em redes Ethernet).
    • Controla o acesso ao meio físico (como CSMA/CD em redes Ethernet).

  2. Camada de Rede:

    • Responsável pelo roteamento de dados entre diferentes redes.
    • Utiliza endereços lógicos (por exemplo, endereços IP) para encaminhar pacotes de dados entre roteadores.
    • Exemplos de protocolos nesta camada incluem o IPv4 (Internet Protocol version 4) e o IPv6.

  3. Camada de Transporte:

    • Gerencia a comunicação ponto a ponto entre dispositivos finais.
    • Fornece controle de fluxo, segmentação e remontagem de dados.
    • Dois protocolos principais nesta camada são o TCP (Transmission Control Protocol), que é orientado à conexão e confiável, e o UDP (User Datagram Protocol), que é não orientado à conexão e mais leve.

  4. Camada de Aplicação:

    • Fornece interfaces para que os aplicativos possam utilizar os serviços de rede.
    • Inclui protocolos como HTTP (Hypertext Transfer Protocol), SMTP (Simple Mail Transfer Protocol), FTP (File Transfer Protocol) e outros protocolos de aplicação.

Protocolos OSI

  • Ethernet (IEEE 802.3): Um protocolo de comunicação amplamente utilizado em redes locais (LANs).
  • USB (Universal Serial Bus): Utilizado para a conexão de dispositivos periféricos em computadores.

  • Point-to-Point Protocol (PPP): Um protocolo comumente usado para estabelecer uma conexão direta entre dois pontos.
  • IEEE 802.1Q (VLAN): Protocolo que permite a segmentação de uma rede em VLANs (Redes Locais Virtuais).

  • Internet Protocol (IP): Responsável pelo endereçamento e roteamento de pacotes na Internet.
  • Internet Control Message Protocol (ICMP): Usado para comunicação entre dispositivos de rede, especialmente para relatórios de erro.

  • Transmission Control Protocol (TCP): Fornecendo comunicação confiável e orientada à conexão.
  • User Datagram Protocol (UDP): Oferecendo comunicação não confiável e de baixa latência.

  • NetBIOS (Network Basic Input/Output System): Usado para comunicação entre aplicativos em uma rede.
  • Session Initiation Protocol (SIP): Comumente utilizado para iniciar, modificar e encerrar chamadas de voz e vídeo pela Internet.

  • Hypertext Transfer Protocol (HTTP): Utilizado para a transferência de dados na World Wide Web.
  • Secure Sockets Layer (SSL) / Transport Layer Security (TLS): Fornecendo segurança na comunicação pela Internet.

  • Simple Mail Transfer Protocol (SMTP): Utilizado para enviar e-mails.
  • File Transfer Protocol (FTP): Usado para a transferência de arquivos pela rede.

Protocolos TPC/IP

  • Ethernet (IEEE 802.3): É um protocolo amplamente utilizado para comunicação de dados em redes locais (LANs). Ele opera na camada de link e fornece os meios físicos para a transmissão de dados.
  • Point-to-Point Protocol (PPP): É um protocolo de camada de link usado para estabelecer uma conexão direta entre dois nós de uma rede. É frequentemente utilizado em conexões de linha discada.

  • Internet Protocol (IP): Este protocolo é essencial para o funcionamento da Internet. Ele fornece o endereçamento e o roteamento dos pacotes de dados entre dispositivos em redes diferentes.
  • Internet Control Message Protocol (ICMP): Usado para enviar mensagens de controle e erros, como mensagens de "ping" para verificar a conectividade de rede.

  • Transmission Control Protocol (TCP): Um protocolo orientado à conexão que fornece entrega confiável de dados, controle de fluxo e retransmissão de pacotes perdidos. É usado em aplicações que exigem uma entrega precisa e ordenada de dados.
  • User Datagram Protocol (UDP): Um protocolo de transporte mais leve e sem conexão, que é adequado para aplicações que podem tolerar alguma perda de dados, como transmissões de áudio e vídeo em tempo real.

  • Hypertext Transfer Protocol (HTTP): Usado para a comunicação na World Wide Web. É o protocolo por trás da transmissão de dados na web, incluindo a exibição de páginas da web.
  • File Transfer Protocol (FTP): Utilizado para a transferência de arquivos entre sistemas. Permite o download e upload de arquivos de e para servidores FTP.
